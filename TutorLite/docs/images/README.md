# README images

The project README references these screenshots by relative path. Drop PNGs with
exactly these names into this folder and they'll render on the GitHub repo page and in
clones. Until you add a file, that spot shows a broken-image icon, so add them before
pushing (or temporarily comment out the matching `<img>`/`![]()` line in the README).

| File | Where it shows | Suggested shot |
| --- | --- | --- |
| `hero.png` | Top banner | An annotated note in Reading view: a highlighted passage with an open margin comment card and its dotted connector. The signature view. |
| `review.png` | Screenshots grid | A margin card showing the agent's review under your note. |
| `srs.png` | Screenshots grid | The "Review due cells" modal grading a memory cell (Again / Hard / Good / Easy). |
| `notebook.png` | Screenshots grid | A generated `Notebook/` page with dated links chaining notebook → annotation → source. |
| `translate.png` | Screenshots grid | An inline gloss inserted after a foreign word/phrase (`Alt+T`). |

Guidelines: PNG for crisp text; resize to roughly 1600px wide and a few hundred KB so the
page loads fast; pick a clean theme; always keep the `alt` text in the README accurate.
For a short demo, an animated GIF works in the same `<img>`/`![]()` slot.
